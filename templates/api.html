Headless API documentation

Script reproducible AutoDock Vina package preparation.

AutoDock-Vina PrepServer exposes a staged JSON/form API for creating docking workspaces, uploading receptors, saving docking centers, preparing receptor files, uploading ligands, building downloadable packages, and retrieving artifacts.

Use browser workflow {% if repository_url is defined %} View GitHub repository {% endif %}

API design

The API is intentionally staged instead of being a single black-box endpoint. Each step produces a visible state: workspace creation, receptor intake, center definition, protein preparation, ligand upload, package build, and artifact download. This makes docking-preparation failures easier to diagnose.

Base URL

Use the deployment domain as the base URL. For the public site, examples use: 

https://autodockvina.com

Authentication note

Public deployments may run in public-access mode. Private deployments can enable authentication through server configuration. API behavior may depend on how the instance is deployed.

Response format

Versioned API endpoints generally return JSON with this structure: 

{
  "ok": true,
  "data": {},
  "warnings": []
}

Data caution

Do not upload confidential receptor structures, proprietary ligand libraries, or sensitive project data to a public deployment unless the instance is explicitly configured for that purpose.

Quick API flow

Typical scripted docking-preparation sequence.

A complete scripted workflow usually follows this order:

1

Create a workspace

Create an isolated job workspace that stores receptors, centers, ligands, logs, and package artifacts.

POST /api/v1/workspaces
2

Add receptors

Upload receptor files or fetch a receptor by PDB ID.

POST /api/v1/workspaces/<jobname>/receptors/upload
POST /api/v1/workspaces/<jobname>/receptors/fetch
3

Resolve and save docking centers

Resolve a center from XYZ coordinates, ligand/residue selection, or supported receptor-based methods, then save it.

POST /api/v1/workspaces/<jobname>/centers/resolve
POST /api/v1/workspaces/<jobname>/centers/save
4

Prepare receptors

Generate receptor preparation outputs and logs for downstream package assembly.

POST /api/v1/workspaces/<jobname>/prep/start
GET  /api/v1/workspaces/<jobname>/prep/status
5

Upload ligands

Upload SDF, SMILES, SMI, CSV, ZIP, or folder-based ligand inputs where enabled.

POST /api/v1/workspaces/<jobname>/ligands/upload
GET  /api/v1/workspaces/<jobname>/ligands
6

Build and download package

Build the ZIP package, list available artifacts, and download the generated package.

POST /api/v1/workspaces/<jobname>/build
GET  /api/v1/workspaces/<jobname>/artifacts
GET  /api/v1/workspaces/<jobname>/download
Endpoint reference

Available API endpoints.

Method Endpoint Purpose
GET /api/v1/health Check service health, API version, public mode, and temporary workspace root.
POST /api/v1/workspaces Create a new workspace or reuse a named workspace when supported.
GET /api/v1/workspaces/<jobname> Read workspace state.
GET /api/v1/workspaces/<jobname>/summary Read workflow summary, receptor status, center rows, ligand state, and artifacts.
POST /api/v1/workspaces/<jobname>/receptors/upload Upload receptor files by single file, ZIP, or folder-style form upload.
POST /api/v1/workspaces/<jobname>/receptors/fetch Fetch a receptor by PDB ID where supported.
GET /api/v1/workspaces/<jobname>/receptors List registered receptors in a workspace.
POST /api/v1/workspaces/<jobname>/centers/resolve Resolve a docking center from JSON payload input.
POST /api/v1/workspaces/<jobname>/centers/save Save a docking center and box size for a receptor.
GET /api/v1/workspaces/<jobname>/centers List saved docking centers.
POST /api/v1/workspaces/<jobname>/prep/start Start receptor preparation/conversion for the workspace.
GET /api/v1/workspaces/<jobname>/prep/status Read receptor-preparation status and logs.
POST /api/v1/workspaces/<jobname>/ligands/upload Upload ligand files, ZIPs, or folder-style ligand inputs.
GET /api/v1/workspaces/<jobname>/ligands List uploaded ligand files and ligand metadata.
POST /api/v1/workspaces/<jobname>/build Build a portable or deployment-configured docking package.
GET /api/v1/workspaces/<jobname>/artifacts List generated ZIP artifacts.
GET /api/v1/workspaces/<jobname>/download Download the latest or requested artifact from a workspace.
Copy-paste examples

Example requests with curl.

Health check

curl https://autodockvina.com/api/v1/health

Create workspace

curl -X POST https://autodockvina.com/api/v1/workspaces \
  -H "Content-Type: application/json" \
  -d '{"workspace_name":"demo-docking-job"}'

Fetch receptor by PDB ID

curl -X POST https://autodockvina.com/api/v1/workspaces/demo-docking-job/receptors/fetch \
  -H "Content-Type: application/json" \
  -d '{"pdb_id":"3EKY","chains":"A"}'

Upload receptor file

curl -X POST https://autodockvina.com/api/v1/workspaces/demo-docking-job/receptors/upload \
  -F "mode=single" \
  -F "file=@receptor.pdb"

Save XYZ docking center

curl -X POST https://autodockvina.com/api/v1/workspaces/demo-docking-job/centers/save \
  -H "Content-Type: application/json" \
  -d '{
    "method":"xyz",
    "receptor":"receptor.pdb",
    "center":[10.5, 22.1, -3.4],
    "size":20
  }'

Start receptor preparation

curl -X POST https://autodockvina.com/api/v1/workspaces/demo-docking-job/prep/start \
  -H "Content-Type: application/json" \
  -d '{
    "remove_hets":"all",
    "remove_chains":"",
    "altloc":"collapse"
  }'

Check preparation status

curl https://autodockvina.com/api/v1/workspaces/demo-docking-job/prep/status

Upload ligand file

curl -X POST https://autodockvina.com/api/v1/workspaces/demo-docking-job/ligands/upload \
  -F "mode=single" \
  -F "file=@ligands.sdf"

Build package

curl -X POST https://autodockvina.com/api/v1/workspaces/demo-docking-job/build \
  -H "Content-Type: application/json" \
  -d '{
    "package_mode":"portable",
    "vina_poses":20,
    "confgen_poses":64
  }'

Download artifact

curl -L -o docking_package.zip \
  https://autodockvina.com/api/v1/workspaces/demo-docking-job/download
Payload notes

Common request fields.

Workspace creation

Useful fields:

{
  "workspace_name": "demo-docking-job",
  "reuse": false
}

Receptor fetch

Useful fields:

{
  "pdb_id": "3EKY",
  "chains": "A"
}

XYZ center save

Useful fields:

{
  "method": "xyz",
  "receptor": "receptor.pdb",
  "center": [10.5, 22.1, -3.4],
  "size": 20
}

Preparation options

Useful fields:

{
  "remove_hets": "all",
  "remove_chains": "",
  "altloc": "collapse"
}

Package build

Useful fields:

{
  "package_mode": "portable",
  "vina_poses": 20,
  "confgen_poses": 64,
  "csv_smiles_col": "SMILES",
  "csv_id_col": "ID"
}

LSF package mode

Useful fields when scheduler-oriented packaging is enabled:

{
  "package_mode": "lsf",
  "workers": 16,
  "queue": "hihg",
  "project": "brd",
  "mem_per_core": 2000,
  "vina_path": ""
}

Important workflow constraints

Package generation requires receptors, saved docking centers, prepared receptor outputs, and uploaded ligands. CSV ligand workflows require a SMILES column before building. Public deployments may restrict package modes or external executables.
